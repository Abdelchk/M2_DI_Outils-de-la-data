# Guide de contribution

## Pour les étudiants

### Comment soumettre vos solutions

1. **Forker le dépôt** sur GitHub
2. **Cloner votre fork** localement
3. **Créer une branche** pour votre travail
4. **Travailler sur l'exercice** dans le dossier `solutions/votre-nom/`
5. **Commit et push** vos changements
6. **Créer une Pull Request**

### Structure de soumission

Pour chaque exercice, créez un dossier avec votre nom :

```
exercices/exercice-XX/
└── solutions/
    └── votre-nom/
        ├── solution.py (ou autres fichiers)
        ├── resultats.md
        └── README.md (optionnel)
```

### Standards de code

- Code commenté et lisible
- Noms de variables explicites
- Fonctions bien documentées
- Tests si possible

## Pour les contributeurs

Les contributions sont les bienvenues ! Merci de :

1. Ouvrir une issue pour discuter des changements majeurs
2. Créer une branche pour votre feature
3. Suivre les standards de code
4. Ajouter des tests si applicable
5. Mettre à jour la documentation
